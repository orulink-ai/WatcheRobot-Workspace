边界速查

ESP32 相关

源码烧录用 watche-s3-flash-monitor；release ZIP 用 watche-s3-release-binary-flash；wrapper 失败后才看 watcher-esp32s3-build-flash。

双 MCU 相关

只有 ESP32 和 STM32 都接入、需要联调日志或 UART2 握手诊断时，用 watche-dual-mcu-bringup。

团队本机配置

端口、设计盘符、ESP-IDF Python 环境等本机事实不要写进 skill；使用 .codex/local、环境变量或运行时参数。

治理类

创建 skill 用 skill-authoring；评估和整理现有 skill 用 skill-evaluation；交付代码阶段用 stage-delivery。

本次规范化结果

  • 默认 ESP32 设备别名统一为 esp32-s3，STM32 设备别名统一为 stm32-f103。
  • 项目 skills 不再默认依赖个人绝对路径；统一使用 <workspace-root>、<esp32-root>、环境变量或参数。
  • ESP32 普通烧录和底层排障分离，减少 watche-s3-flash-monitor 与 watcher-esp32s3-build-flash 的选择冲突。
  • 设计动画导入脚本支持从当前工作区或 WATCHE_S3_FIRMWARE_ROOT 自动发现固件根目录。
