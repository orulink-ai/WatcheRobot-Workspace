# 新人检查清单

## 阶段 0：背景确认

- 已说明负责方向。
- 已说明是否有硬件台架。
- 已说明本次目标：只读导览、App/桌面端/服务端练习、构建烧录练习、小改动练习。

## 阶段 1：项目地图

新人应能说明：

- meta repo 管理什么。
- `WatcheRobot_app` 管理什么。
- `WatcheRobot_client` 管理什么。
- `WatcheRobot_server` 管理什么。
- `WatcheRobot_esp32` 管理什么。
- `WatcheRobot_stm32` 管理什么。
- 为什么跨 App、桌面端、服务端和固件的修改要分仓提交并说明配套关系。
- 哪些个人文件或临时文件不能提交。

## 阶段 2：工作流入口

新人应能根据任务选择入口：

- yarn 指令、脚本和 skill 的使用边界。
- App 启动、测试和平台运行入口。
- 桌面端开发、类型检查和构建入口。
- 服务端启动和测试入口。
- ESP32 构建/烧录/监控。
- STM32 构建/烧录。
- 双 MCU 联调。
- 动画资源导入。
- release 二进制烧录。
- 阶段性交付。
- PR 鲁棒性审查。

## 阶段 2.5：调试工具导览

新人应能说明：

- 什么时候优先调用 skill。
- 什么时候查看 `package.json` scripts 或 workspace 脚本。
- 如何判断脚本是否会烧录设备、覆盖资源或生成发布产物。
- 遇到调试目标时，如何从自然语言问题映射到 yarn 指令、脚本或 skill。

## 阶段 3：实操路径

### 只读导览

适合暂时没有硬件或只想了解项目的新人。完成标准：能说清楚仓库边界、关键 skill、常见风险和下一步实操路径。

### 构建烧录练习

适合有硬件台架的新人。完成标准：在对应专项 skill 带领下完成一次 ESP32 或 STM32 构建/烧录/监控，并能解释每一步影响了哪个设备或文件。

### App/桌面端/服务端练习

适合负责客户端或服务端的新人。完成标准：完成一次对应仓库的启动或测试，能说明服务端、桌面端、App 与硬件链路的边界。

### 小改动练习

适合准备开始贡献代码的新人。完成标准：完成一个低风险改动，说明测试或验证目标，按仓库边界提交，并准备阶段性交付。

## 结束前必须追问

使用以下问题结束 onboarding：

```text
现在还有哪些地方不理解、不确定，或者你觉得文档/流程没讲清楚？
可以包括环境配置、构建烧录、仓库结构、脚本用途、提交流程、调试方法、硬件联调等。
```

## 完成标准

- 新人知道下一步该调用哪个 skill 或执行哪个工作流。
- 新人知道哪些操作有硬件或仓库风险。
- 新人知道不理解时应该如何提问和如何把问题沉淀回 onboarding。
- Agent 已判断本次问题是否需要更新 onboarding skill 或 references。
