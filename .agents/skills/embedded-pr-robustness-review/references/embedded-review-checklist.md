# 嵌入式审查清单

在审查嵌入式 C/C++ 和 RTOS 相关 PR 时，用这份清单聚焦稳定性风险。

## 内存与生命周期

- 检查每条分配路径的所有权、部分失败时的清理、重复释放风险，以及提前返回时的泄漏。
- 检查任务、回调、解析器、显示、摄像头和音频路径中的栈使用。
- 检查缓冲区长度、分配或拷贝前的整数溢出、字符串结尾、payload 长度可信度和对齐假设。
- 把外部输入都视为不可信：UART、BLE、Wi-Fi、WebSocket、SD 卡、摄像头、生成资产、JSON，以及 NVS/OTA 元数据。
- 当缓冲区跨硬件边界使用时，确认 DMA、PSRAM 和 cache 约束被满足。

## 并发与 RTOS

- 找出每个会被多个任务、ISR、回调、定时器、事件循环或驱动同时访问的共享状态。
- 确认锁同时覆盖读写路径；除非本地已有明确模式，否则不要在持锁时调用阻塞 API。
- 检查 ISR-safe API 使用、队列发送方式、定时器回调限制和任务通知所有权。
- 留意丢失唤醒、过期缓存状态、优先级反转、无界等待，以及没有延时的重试循环。
- 确认对象释放不会和回调或正在执行的工作并发冲突。

## 协议与解析器

- 验证畸形包、截断包、重复包、延迟包、乱序包和超大包。
- 检查部分读取、粘包、重同步、CRC/鉴权、版本字段、大小端转换，以及序列号/引用语义。
- 确认未知命令默认安全失败，不会破坏状态。
- 确认解析错误有足够可观测性便于调试，同时有频率限制，避免日志风暴。

## 硬件资源

- 检查 GPIO、I2C、SPI、UART、PWM、ADC、DMA 的所有权，以及组件之间的资源冲突。
- 确认 init/deinit 顺序、pin mux、总线速度、上拉、片选行为、复位线和电源轨。
- 验证外设缺失、忙碌、响应慢或返回瞬时错误时的降级行为。
- 检查长时间硬件操作、Flash/SD 写入、显示刷新、摄像头采集和网络重连对 watchdog 的影响。

## 状态机与恢复

- 枚举 PR 修改到的状态和转换，检查非法转换处理和幂等性。
- 确认重连、重试、退避、失败后的安全默认值，以及部分初始化失败后的恢复行为。
- 检查启动顺序、服务依赖，以及某个子系统被禁用或失败时的行为。
- 确认需要调用方采取行动的错误会向上传播；只有在有注释或日志说明时才有意吞掉错误。

## 存储、OTA 与生成资产

- 验证 partition、NVS、SD、SPIFFS、资产 manifest 和生成二进制格式的边界与兼容性。
- 检查原子性、回滚、版本迁移和掉电行为。
- 确认生成文件与运行时解析器在字节序、尺寸、数量、限制和可选字段上保持一致。

## 日志与诊断

- 检查日志包含足够的现场诊断上下文：状态、错误码、命令、序列号、大小和资源 id。
- 避免高频循环和解析错误路径中的日志刷屏。
- 确认 metrics/counters 是有意重置的，并且不会溢出成误导性数值。

## 对外接口

- 对 header/API 变更，检查所有调用方和默认值。
- 确认所有权和线程模型约定已经被文档、命名或本地模式清楚表达。
- 检查 wire format、持久化数据、生成资产和远端客户端的向后兼容性。
