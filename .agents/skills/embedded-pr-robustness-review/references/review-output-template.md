## 缺失验证

- 列出合入前应补充的构建、host、硬件、压力或日志证据。
- 说明每个缺失检查覆盖的是哪个变更风险域。
- 当前证据足够时写 `未发现`。

## 已检查但未发现问题的区域

- 简要列出已检查且未发现问题的重要区域，例如内存生命周期、RTOS 并发、解析器边界、硬件资源、启动顺序或存储兼容性。
- 保持简短真实，不要声称覆盖了未实际检查的区域。
