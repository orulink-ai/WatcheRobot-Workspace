## Acceptance Principles

- Unknown License, community entrance, Demo, maintainer, hardware scope, and roadmap dates must stay in [Open Questions](open-questions.md) or [Owner Decision Record](owner-decision-record.md).
- Locally verifiable work must pass `scripts/check-open-source-readiness.ps1`.
- Plan structure must pass `scripts/test-delivery-plan-structure-contract.ps1` so checkpoints, target table, todo list, sub-agent strategy, TDD, and self-scoring rules are not dropped.
- Sub-agent work orders must pass `scripts/test-sub-agent-work-orders.ps1` so WO-01~WO-07 keep inputs, allowed actions, forbidden actions, verification commands, escalation rules, deliverables, and self-score notes.
- Owner decisions must pass `scripts/test-owner-decision-record.ps1` and `scripts/test-owner-decision-quality-fixtures.ps1` so OQ-001~OQ-009 remain complete, state-consistent, traceable, and non-future-dated when closed.
- Owner decision handoff must pass `scripts/test-owner-decision-brief.ps1` so all 9 owner questions map to blocked gates, evidence types, and files to update after approval.
- Launch evidence templates must pass `scripts/test-launch-evidence-templates.ps1` so empty templates cannot be treated as passed evidence.
- Launch evidence request pack must pass `scripts/test-launch-evidence-request-pack.ps1` so every gate and owner question has requestable evidence fields.
- Uncertainty governance must pass `scripts/test-uncertainty-governance-contract.ps1` so unresolved items continue to use owner questions, owner decisions, or explicit pending markers.
- GitHub remote web snapshots are fallback evidence only and must pass `scripts/test-github-web-snapshot-contract.ps1`.
- GitHub Actions readiness workflow must pass `scripts/test-open-source-ci-workflow.ps1`.
- Markdown local links and heading anchors must pass `scripts/audit-markdown-links.ps1` and `scripts/test-markdown-link-audit.ps1`.
- Evidence collector coverage must pass `scripts/test-evidence-collector-coverage.ps1`.
- Product naming must pass `scripts/test-product-name-policy.ps1` so public materials use `WatcheRobot`.
- Readiness scoring must pass `scripts/test-readiness-score-contract.ps1` and must not claim 100/100 until all launch gates pass.
- Launch gate closeout plan must pass `scripts/test-launch-gate-closeout-plan.ps1`.
- The Word reference plan must pass `scripts/test-plan-docx-contract.ps1`, and DOCX render prerequisites must be checked through `scripts/audit-docx-render-prerequisites.ps1` plus `scripts/test-docx-render-prerequisites-audit.ps1`; unavailable visual-render tooling is an explicit unavailable gate, not completed visual QA.
- Goal completion audit must pass `scripts/test-goal-completion-audit.ps1` before the active goal can be marked complete.
- Public README must pass `scripts/test-public-readme-contract.ps1`.
- This docs index must pass `scripts/test-docs-index-contract.ps1`.
- Developer onboarding must pass `scripts/test-developer-onboarding-contract.ps1`.
- Workspace submodule references must pass `scripts/test-workspace-submodule-contract.ps1`.
- Release and handoff runbooks must pass `scripts/test-open-source-runbooks.ps1`.
- Publication hygiene must pass `scripts/audit-publication-hygiene.ps1` and `scripts/test-publication-hygiene.ps1`.
- WOS-01~WOS-45 coverage and evidence trace must pass `scripts/test-wos-coverage.ps1` and `scripts/test-wos-evidence-trace.ps1`.
- Launch gate regression tests must pass `scripts/test-open-source-launch-gates.ps1`.
- Public Chinese entry quality must pass `scripts/audit-open-source-text-quality.ps1`.
- Before public launch, [Public Launch Validation Runbook](public-launch-validation.md) must receive clean-machine, hardware, GitHub admin, release artifact, and owner sign-off evidence.
